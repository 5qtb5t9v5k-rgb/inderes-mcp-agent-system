## Per-case audit

### case_001 — comparison routing

**Original failure mode:** 19 of 30+ comparison runs got only
`["quant"]`; LEAD then fabricated "business model" reasoning without
research data.

**Fix:** `_enforce_comparison_floor()` in
`src/inderes_agent/orchestration/router.py`. When
`is_comparison=True`, programmatically expands `domains` to include
`{quant, research, sentiment}` regardless of what Flash Lite emitted.
Five unit tests in `test_router.py` lock the contract.

**Why we treat this as fixed without re-running evals:** the assertion
that failed (`'research' in routing.domains`) is now structurally
guaranteed by the post-classify code path — not by prompt obedience.
The router cannot emit a `is_comparison=true, domains=['quant']` shape
to a downstream consumer; the floor is applied between
`_extract_json` and `return`.

**Residual risk:** if `is_comparison` itself is misclassified (e.g.
the router says `False` for a comparison query), the floor never
fires. Eval would catch this via the routing-companies assertion.

### case_002 — päättely schema dead

**Original failure mode:** 96 runs `paattely_kind=empty`, 55 `prose`,
**zero structured**. Spec required structured `{disagree, resolution,
uncertain, skipped}` but LEAD always emitted prose.

**Fix:** `80c6fd0 fix(paattely): prose→slot lift + stronger
conflict-naming rule`. The fix lifts free-text rationale into the
structured slot grid, so even when LEAD writes prose the post-process
extracts disagree/resolution/uncertain/skipped where present.

**Why we still mark this 🟡:** unlike case_001, this fix is in
synthesis processing rather than a deterministic gate. The `paattely_kind`
field comes from the run-log, populated by the LEAD prompt + post-
processing. A live run is needed to confirm the kind transitions from
`prose` to `structured` (or that the lift produces the equivalent).

**Action:** re-run case_002 (single-company valuation query —
"tee arvonmääritys Sampolle") and check `paattely_kind == 'structured'`.

### case_004 — search robustness (Vincit fabrication)

**Original failure mode:** Vincit query (not in MCP catalog) ran
308 s with subagent errors; LEAD fabricated entire analysis. Judge:
*"complete failure, hallucinated entire analysis"* — score 1/5.

**Fix:** `870749a fix(orchestration): fabrication guard + no-data
path — fix #2 from Tier 1 baseline`. Hard MCP-gate: any subagent that
emitted zero `get-fundamentals` (or domain equivalent) calls is
flagged as fabricated; LEAD then surfaces "yhtiötä ei löytynyt
MCP-tietokannasta" rather than synthesising on hallucinated data.

Hard limit `max_workflow_duration=180` (Wk 1 #3) prevents the 308 s
loop separately.

**Why 🟡:** the hard `duration_s < 90` assertion in golden.yaml
should now pass thanks to the workflow cap, but the `subagent_errors
> 0 OR 'ei löyty' in synthesis` branch needs live verification. The
fabrication guard is observed via `subagent_errors > 0` — a live
re-run should produce a green case.
